# R Package Development

Flode is the team's meta-package. Individual modules (`reach.io`, `reach.hydro`, etc.) are developed as standalone packages following these standards. The same standards apply to any team R package intended for Tier 3 or Tier 2 use.

---

## 1. Package structure

```
reach.hydro/
├── DESCRIPTION
├── NAMESPACE
├── README.md
├── NEWS.md              # changelog: one entry per version
├── R/
│   ├── calc_flood_peak.R
│   ├── fit_pdm.R
│   └── utils-internal.R
├── tests/
│   └── testthat/
│       ├── test-calc_flood_peak.R
│       └── test-fit_pdm.R
├── man/                 # generated by roxygen2 -- do not edit manually
├── vignettes/           # long-form documentation
└── renv.lock
```

One file per exported function (or per closely related group of functions). Test files mirror source files with a `test-` prefix.

## 2. DESCRIPTION file

```
Package: reach.hydro
Title: Hydrological Calculations for Flood Forecasting and Warning
Version: 0.1.0
Authors@R: person("Jonathan", "Payne", email = "jonathan.payne@environment-agency.gov.uk",
  role = c("aut", "cre"))
Description: Core hydrological calculations for the EA Flood Forecasting and Warning
    team. Implements PDM rainfall-runoff modelling, FEH flood frequency methods,
    ARMA error correction, and standard flow statistics. Part of the Flode ecosystem.
License: GPL (>= 3)
Encoding: UTF-8
Roxygen: list(markdown = TRUE)
RoxygenNote: 7.x.x
Depends:
    R (>= 4.3.0)
Imports:
    collapse (>= 2.0.0),
    data.table (>= 1.15.0),
    here (>= 1.0.0)
Suggests:
    testthat (>= 3.0.0),
    covr
```

**Dependency rules:**
- `Imports`: packages required at runtime. Declare every non-base package used.
- `Suggests`: packages needed only for tests or vignettes.
- `Depends`: use only for R version and packages that must be attached (rare).
- Never declare tidyverse packages in `Imports` for a Tier 3 module.
- Use specific version lower bounds based on the features you actually use.

## 3. roxygen2 documentation

All exported functions must have full roxygen2 documentation for Tier 3. Tier 2 requires documentation for key functions at minimum.

```r
#' Calculate median annual flood (QMED) from catchment descriptors
#'
#' Estimates QMED using the FEH statistical method from AREA, BFI, and SAAR.
#' Suitable for gauged catchments; for pooling group analysis see [pool_qmed()].
#'
#' @param catchment A [FlodeCatchment] object containing the required catchment
#'   descriptors.
#' @param adjust Logical. If `TRUE`, apply the donor catchment adjustment where
#'   a suitable donor is available. Default `FALSE`.
#'
#' @return A double. Estimated QMED in m³/s.
#'
#' @examples
#' thames <- FlodeCatchment(
#'   gauge_id = "4206TH",
#'   area_km2 = 9948.0,
#'   bfi      = 0.47,
#'   saar_mm  = 706.0
#' )
#' calc_qmed(thames)
#'
#' @export
calc_qmed <- function(catchment, adjust = FALSE) {
  ...
}
```

### Key roxygen2 tags

| Tag | Use |
|---|---|
| `@param` | Every argument; describe type, expected range, and default if non-obvious |
| `@return` | Describe the return object, its type, and its structure |
| `@examples` | Runnable example using small embedded data; no external file dependencies |
| `@export` | All functions intended for users |
| `@keywords internal` | Internal helpers not exported |
| `@seealso` | Link to related functions |
| `@references` | FEH, scientific papers, or governance docs |

## 4. Internal vs exported functions

Prefix internal helpers with `.` to signal non-export intent and exclude them from the namespace.

```r
# Internal: not in NAMESPACE, not accessible to users
.validate_params <- function(params) {
  stopifnot(all(c("b", "cmax", "kg") %in% names(params)))
}

# Exported: in NAMESPACE, user-accessible
#' @export
calc_qmed <- function(catchment) { ... }
```

## 5. Dependency management with renv

Every package repository must use `renv`.

```r
renv::init()        # initialise for a new package
renv::snapshot()    # after adding or updating any dependency
renv::restore()     # first step after cloning
renv::status()      # verify lockfile is current before merge
```

`renv.lock` must be committed alongside any change to `DESCRIPTION`. A fresh install via `renv::restore()` must reproduce the environment with no undeclared dependencies.

## 6. Testing with testthat

### Structure

```r
# tests/testthat/test-calc_flood_peak.R

test_that("calc_flood_peak returns expected value for known catchment", {
  catchment <- FlodeCatchment(
    gauge_id = "TEST01",
    area_km2 = 100.0,
    bfi      = 0.4,
    saar_mm  = 800.0
  )
  result <- calc_flood_peak(catchment)
  expect_equal(result, 42.7, tolerance = 0.1)
})

test_that("calc_flood_peak errors on non-positive area", {
  catchment <- FlodeCatchment(
    gauge_id = "TEST01",
    area_km2 = -1.0,
    bfi      = 0.4,
    saar_mm  = 800.0
  )
  expect_error(calc_flood_peak(catchment))
})

test_that("calc_flood_peak handles NA flow correctly", {
  # ... edge case
})
```

### Coverage requirements

| Tier | Coverage | Enforcement |
|---|---|---|
| Tier 3 | 70% line coverage via `covr` | Blocks merge if below threshold |
| Tier 2 | Tests recommended | Advisory |
| Tier 1 | Inline comments sufficient | None |

```r
# Check coverage locally
library(covr)
package_coverage()
```

Run `devtools::check()` as the definitive pre-submission quality gate. It catches namespace problems, undeclared imports, and broken examples that `lintr` and `testthat` alone do not:

```r
# Clean check required before any Tier 3 merge request or Flode promotion
devtools::check()
```

### Test design rules

- Cover expected outputs, edge cases (empty input, `NA` values, out-of-range parameters), and known failure modes.
- Use small, embedded data fixtures. No external file dependencies in tests.
- One `test_that()` block per behaviour, not per function.
- Tests must pass in a fresh R session after `renv::restore()`.

## 7. Flode promotion path

To promote a function from `experimental/` into a Flode module:

1. Develop in `experimental/` as Tier 1 with full header block.
2. Peer review to Tier 2 standard: header complete, logic documented, key edge cases handled.
3. Open a GitHub issue to the Flode Steward describing the function, its target module, and its test status.
4. Write `roxygen2` documentation covering `@param`, `@return`, `@examples`, and `@export`.
5. Achieve 70% line coverage via `covr`.
6. Pass `lintr` with no errors.
7. Steward reviews, merges, increments the module version, and updates `NEWS.md`.

Do not add functions to Flode directly. All contributions go through the Steward.

## 8. Version numbering

Follow semantic versioning: `MAJOR.MINOR.PATCH`.

- `PATCH`: bug fixes; no API changes.
- `MINOR`: new functions or arguments added; existing API unchanged.
- `MAJOR`: breaking changes to the API.

Update `NEWS.md` for every version increment. Format:

```md
# reach.hydro 0.2.0

## New functions
- `calc_kge()`: Kling-Gupta Efficiency with decomposed components.

## Changes
- `calc_qmed()`: added `adjust` argument for donor catchment correction.

## Bug fixes
- `fit_pdm()`: fixed off-by-one in warmup period calculation (#14).
```
